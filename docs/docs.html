Waves Payment

Introdução

Está é a documentação do Plugin para wooocommerce Waves Payment, que permite a compra de produtos em lojas virtuais wooocommerce por meio de Assets existentes na blockchain da Waves. Nele é possivel comprar com qualquer token que esteja configurado corretamente para a opção de compra. Abaixo você irá aprender a como ele funciona, as oconfigurações iniciais, adição de Assets existentes na blockchain, configurar banco de dados SQL, além de detalhes técnicos sobre como ele funciona e como fazer a manutenção no mesmo.

O que é Waves Payment

Waves Payment é um plugin wordpress criado para ser usado como forma de pagamento para websites que utilizem o wooocommerce, de forma a tornar possivel a compra de produtos com assets da blockchain da Waves, dando a possibilidade de configurar até 10 Assets para a escolha de compra.

Como funciona?

Após o administrador do website que irá usa-lo configurar o wooocommerce corretamente, finalizar a adição e ativação do plugin no wordpress, será necessário fazer as configurações necessárias, apresentadas abaixo, para o funcionamento correto do plugin. Quando configurado corretamente, assim que o usuário for para a página de finalização, irá aparecer a opção de escolha de forma de pagamento, basta ele selecionar Waves Payment, ou o nome que tiver configurado como "title", e escolher um dos Assets a serem utilizados. Após a seleção, é gerado um qr code com o endereço de depósito para a realização do pagamento, juntamente com um crônometro marcando o tempo ao qual ele tem para realizar o pagamento, caso esse tempo expire a página será recarregada. Se algum valor for depositado no endereço gerado após o tempo ser expirado, basta o cliente entrar em contato com o responsável pelo website, informando o ocorrido juntamente com o endereço ao qual foi depositado o valor. Caso algum valor não tenha sido repassado para a conta principal pelo endereço gerado, basta fazer uma consulta no banco de dados com o endereço, usar a seed phrase desse endereço na Exchange e fazer o reembolso. Para mais assuntos como esse vá até o final da página, em "Problemas e resolução".

Config.

Como configurar?

Para configurar o plugin para wooocommerce waves-payment, vá até plugins, em "waves-payment" clique em "Configure" e vá na categoria "Config.". Siga as etapas descritas abaixo sem pular nenhuma. Caso fique algo faltando, o plugin não irá funcionar corretamente podendo não iniciar, dar erro no wordpress, erro no javascript ou não funcionar adequadamente levando a perda de valores para os usuários em geral.

1. Titulo, Descrição e Instruções

Para configurar o nome ao qual ficará como apresentação para a seleção de fomas de pagamento basta mudar o campo designado como "Title", para configurar a descrição a qual ficará sobre a forma de pagamento basta mudar o campo designado como "Description", para configurar as instruções que serão dadas para o usuário de como utilizar a forma de pagamento basta mudar o campo designado como "Instructions". Para fazer essas alterações é necessário estar na página de configuração do plugin no painel admin, indo em plugins, em "Waves-payment" e clicando em "Configure".

2. Endereço Waves, Modo de operação e Moeda Fiduciaria

Para configurar o endereço Waves ao qual irá receber os valores pagos pelos clientes que usarão o waves-payment como forma de pagamento, basta alterar o campo "Endereço Waves" com um endereço administrado pela loja. Para configurar o modo de operação da Blockchain em Mainnet ou Testnet, basta ir no campo "Modo de operação" e marcar ou desmarcar o checkbox "Ativar modo TESTNET", caso esteja marcado o modo é na rede de TESTNET, do contrário, se estiver desmarcado é a "MAINNET" que esta em uso. Para configurar a moeda a qual será usada para a conversão de valores com base no dolar, basta ir até o campo "Moeda fiduciaria a ser utilizada" e alterar para a moeda a qual será usada, no qual estão disponivel para utlização o USD, BRL, EUR, RUP, RMB, JPY, ARS e NGN

3. Total de Assets, Query do botão finalizar e Temporizador

Para configurar o total de assets/token a serem utilizados como formas de pagamento aceitas pela loja por meio da blockchain da waves, basta alterar o campo "Total de Assets" para a quantia de assets/tokens, lembrando que não pode ser 0 e não pode ser superior a 10, que estejam configurados por completo e que serão aceitos como forma de pagamento. Esse é opicional, por ser um valor padrão em todos os layouts wordpress com o botão de finalizar compra, mas caso você realize o teste e veja que o valor foi confirmado como recebido, porém não foi dado um "clique" automatico para o função do botão, será necessário configurar o query desse botão para que o clique automatico ocorra, indo em "Query do botão finalizar" e colocando o Query do botão finalizar do layout utilizado, recomendável entrar em contato com o criador do plugin, ou um profissional que saiba de orientação a objeto no DOM com jquery, para lhe auxiliar na configuração. Para configurar o tempo ao qual terá a duração do Temporizador, basta alterar o campo "Temporizador" e colocar o valor total de minutos a serem designados para o mesmo, porém esse valor deve estar em segundos.

4. Testnet Node URL e Mainnet Node URL

Para configurar a URL do node da blockchain da Waves para a rede Testnet, basta alterar o campo "Testnet Node URL" com a URL designada para o mesmo. Para configurar a URL do node da blockchain da Waves para a rede Mainnet, basta alterar o campo "Mainnet Node URL" com a URL designada para o mesmo. A configuraçao da URL já vem de padrão no plugin, mas caso ocorra um erro com o WavesAPI no console, o problema pode ser alguma alteração nelas, ou a ausencia de internet.

Configurações dos Assets

Como configurar?

Para conigurar um Asset/Token da Blockchain da Waves no plugin waves-payment, é necessário, antes de tudo, escolher os Assets/tokens a serem utilizados para isso, lembrando que não podem utrapassar o valor de 10 assets/tokens escolhidos e não pode ser 0. Caso for usar o Waves, não há a necessidade de preencher nada, apenas colocar no ID do asset como Waves e o valor do Sponsorship como 0.001, mas caso queira adicionar um asset/token presente na Waves, sendo próprio ou não, você terá que preencher conforme é dito abaixo.

1. Asset ID, Nome do Asset e Decimais do Token

Para configurar o Nome ao qual será apresentado na escolha de Assets/token pelo usuário ao selecionar Waves-payment como forma de pagamento, você terá que mudar o campo designado como "Nome do Asset" com base no total de assets/tokens configurados para serem utilizados. Para alterar o ID do asset/token a ser utilizado com base na altura em relação ao total de assets/tokens configurados para serem utilizados, você irá mudar o campo designado como "Asset ID" em cada um dos assets/token a serem utilizados. Para alterar os Decimais do Asset/Token que será utilizado como forma de pagamento, basta alterar o campo desginado como "Decimais do Token" com base na altura em relação ao total de assets/tokens configurados para serem utilizados.

2. Fee token Sponsorship e Valor do Asset

Para configurar o Sponsorship e definir um valor para o seu Asset/Token, basta seguir para "Waves Blockchain" em "Detalhes de ajuda ténica" presente nessa página. Logo após definir esses valores iniciais, basta colocar o valor do Sponsorship no campo designado como "Fee token Sponsorship".

Banco de dados SQL

Você deve estar se perguntando "Para que o uso de um banco de dados SQL em um plugin que utiliza a tecnologia da blockchain?". O uso dele, nesse caso, se aplica para armazenar e administrar as contas e seedphrases de endereços de deposito e repasse de valores gerado aleatóriamente para o usuário como forma de qr code. É necessário o armazenamento desses dados por questões de segurança e garantia, para caso o usuário faça o depósito de um asset/token errado ou deposite após o tempo do contador, assim impedindo que ele perca o valor e você não fique no prejuízo. Mais detalhes sobre problemas parecidos presente no final desse website, na parte de "Problemas e Resolução".

Como configurar?

1. SQL Url, Usuário SQL e Senha SQL

Para configurar o seu banco de dados SQL, iremos começar configurando a URL do banco de dados, o usuário e a senha. Para isso vá até "Banco de dados SQL", em "SQL URL", coloque a URL do seu banco de dados, em "usuário sql",coloque o seu usuário de acesso ao banco de daos SQL, em "senha sql", coloque a senha do seu usuário SQL.

2. Nome do Banco de Dados e Tabela

Para configurar o nome do banco de dados, coloque em "banco de dados sql" o nome do seu banco de dados. Para configurar a tabela presente no banco de dados e designada para ser utilizada para isso, altere o campo "tabela banco de dados sql" com o nome da tablela que ficara esses dados.

Waves Blockchain

O plugin para wooocommerce waves-payment utiliza como blockchain principal, a blockchain da Waves para isso. Nela é possivel realizar testes do plugin, sem ter valor real, com a Testnet. Já com a Mainnet, após realizar o test na testnet, é possivel receber e enviar valores reaisjdddd. Possuindo uma velocidade bem superior em relação a outras blockchain quando se trata de tempo de transação de valores, além da facilidade e agilidade no desenvolvimento.

Detalhes de ajuda técnica

Veja abaixo detalhes de ajuda técnica para lhe ajudar na criação de um asset/token, adicionar Sponsorship para seu token e definir um valor inicial para ele.

1. Criando um Asset

Para criar um asset/token na blockchain da waves, acesse waves.exchange, crie uma conta, deposite 1 Waves em sua conta, vá em "Create Token", preencha os campos designados nessa seção, como Nome, descrição, quantia total a ser emitida, número de casa, defina se ele pode ser reemitivel e clique em criar.

2. Adicionando o Sponsorship

Para adicionar o Sponsorship em seu asset/token, vá em "My Issued Tokens", selecione o seu asset/token, clique nos três pontinhos e clique em "Manage Sponsorship". Logo após isso,irá abrir um poup-up explicando sobre o funcionamento do Sponsorship e a quantia a qual você deseja definir como Sponsorship, recomendo que seja quantias que sejam, em valores, equivalente ao preço de 0.001 Waves ou 0.001 em seu asset/token.

3. Pares do mercado e Definição de valores

Para definir um valor para sua moeda, você irá precisar fazer uma negociação inicial dela na exchange. Para isso, vá em Trading > Spot > Others Pairs, desmarque o checkbox de "modo verificado" para "modo comunidade", pesquise pelo nome de seu asset/token, va no par do seu asset/token com o USDN. Defina uma quantia a qual será vendida de seu asset/token no par {seu asset}/USDN, é necessário que as negociações aconteçam somente no par com o USDN por conta do método de conversão e calculo de valores que é utilizado no plugin. Defina uma ordem de venda de seu token em relação ao USDN para definir um preço a ele, depois compre essa ordem de venda para gerar uma transação de negociação na blockchain e assim definir um preço.

Problemas e Resolução

Veja abaixo alguns problemas dos quais você ou seus usuários podem passar e que podem ser resolvidos de forma simples e objetiva.

1. Valor não repassado para a loja

O usuário abre uma reclamação com o suporte da loja contestando de que o produto não foi validado e de que foi pago o valor em criptomoedas. Quando isso acontecer, peça para que ele passe o endereço que foi supostamente gerado, verifique na blockchain se esse endereço possui somente 2 transações, se ele consta no banco de dados sql ou se ele possui apenas uma transação do valor mencionado e também esteja no banco de dados. Caso possua mais de uma transação, o usuário está equivocado ou realizou o deposito no endereço errado. Caso possua 2 transações, consta no banco de dados e a última transação foi para o endereço da loja, provavelmente deve ter sido um "bug" local com o usuário. Caso possua uma transação e consta no banco de dados, provavelmente foi um "bug" local com o usuário. Caso possua duas transações ou uma transação e não consta no banco de dados, o usuário esta equivocado ou passou o endereço errado. Mas caso os fatos se saiam como verdadeiros, basta fazer a pesquisa do endereço no banco de dados e fazer login na exchange com a seed phrase do endereço e enviar o valor para o endereço da loja. "Bugs locais", não necessariamente, significa que foi algo de errado no plugin, pode ter ocorrido por conta da velocidade de internet do usuário ou a ausência da mesma.

2. Valor depositado após o tempo limite

O usuário abre uma reclamação com o suporte da loja contestando de que o valor depositado para o endereço gerado no qr code foi efetuado após o limite de tempo do cronometro e deseja o valor de volta. Para resolver isso, peça que o usuário informe o id da transação e o endereço ao qual foi feito o deposito, faça uma pesquisa no banco de dados com esse endereço informando, caso conste no banco de dados, veja esse endereço no explorer e veja se o ID da primeira transação é igual o ID da transação a qual ele informou. Caso o endereço conste no banco de dados, o ID da primeira transação feita para a conta é igual o ID da transação ao qual ele lhe informou e o endereço dele é igual o endereço que está como o endereço que fez o envio dessa transação, então é possivel efetuar o reembolso de forma segura somente para o endereço que fez o envio. Para isso, basta fazer a pesquisa do endereço no banco de dados e fazer login na exchange com a seed phrase do endereço e efetuar o reembolso.

3. Valor diferente do enviado não reembolsado

No plugin para wooocommerce waves-payment, caso o valor depositado como forma de pagamento exceda ou seja menor que o valor definido como ser pago presente abaixo do qr code, então o valor total dessa transferência mal efetuada será reembolsado, contanto que não seja menor que o valor definido no Sponsorship. Caso não ocorra o reembolso e o usuário faça uma reclamação na loja, faça a mesma validação de segurança usada em "Valor depositado após o tempo limite", caso tudo ocorra de forma veridica, basta fazer uma pesquisa com o endereço no banco de dados, fazer login com a seed phrase na exchange e efetuar o reembolso para o endereço que fez o envio.

4. Asset errado enviado por engano

O usuário abre uma reclamação com o suporte da loja contestando de que efetuou o envio de um asset diferente da opção escolhida por ele como asset/token para pagamento no waves-payment,faça a mesma validação de segurança usada em "Valor depositado após o tempo limite", caso tudo ocorra de forma veridica, basta fazer uma pesquisa com o endereço no banco de dados, fazer login com a seed phrase na exchange e efetuar o reembolso desse asset/token que foi enviado por engano para o endereço que fez o envio.

5. Envio de mais de uma vez do mesmo valor

O usuário abre uma reclamação com o suporte da loja contestando de que efetuou o envio do mesmo valor 2 vezes, faça a mesma validação de segurança usada em "Valor depositado após o tempo limite", porém validando as transações após a primeira e segunda transação, caso tudo ocorra de forma veridica, basta fazer uma pesquisa com o endereço no banco de dados, fazer login com a seed phrase na exchange e efetuar o reembolso do valor que foi enviado pela segunda vez por engano para o endereço que fez o envio.
